# InboxPilot

AI-powered email assistant. Connect your email account and manage it through natural conversation.

## Features

- **Chat-based email management** — search, read, summarize, and send emails via AI chat
- **Any email provider** — works with any IMAP/SMTP server (Gmail, Outlook, corporate mail, etc.)
- **Dual AI support** — choose between Claude (Anthropic) or GPT-4o (OpenAI)
- **Multi-language** — AI detects and responds in the user's language automatically
- **Streaming responses** — real-time SSE streaming for instant feedback
- **Secure** — credentials encrypted with AES-256-GCM at rest
- **Draft confirmation** — AI always shows email drafts before sending

## Tech Stack

- **Next.js 14** (App Router, TypeScript)
- **PostgreSQL + Prisma** (database)
- **NextAuth.js v5** (authentication)
- **imapflow + nodemailer** (email protocols)
- **Anthropic SDK + OpenAI SDK** (AI providers)
- **Tailwind CSS** (styling)

## Quick Start

```bash
cd inboxpilot

# Run setup script (installs deps, generates secrets)
chmod +x setup.sh && ./setup.sh

# Edit .env with your database URL and API keys
nano .env

# Create database tables
npx prisma db push

# Start development server
npm run dev
```

Open http://localhost:3000, create an account, connect your email in Settings, and start chatting.

## Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `DATABASE_URL` | Yes | PostgreSQL connection string |
| `NEXTAUTH_SECRET` | Yes | Auth session secret (auto-generated by setup.sh) |
| `ENCRYPTION_KEY` | Yes | 32-byte hex key for credential encryption (auto-generated) |
| `ANTHROPIC_API_KEY` | One of these | Anthropic API key for Claude |
| `OPENAI_API_KEY` | One of these | OpenAI API key for GPT-4o |

## How It Works

1. User sends a message in the chat
2. Backend streams the message to Claude or GPT-4o with email tool definitions
3. AI decides which tools to call (search, read, send, etc.)
4. Backend executes IMAP/SMTP operations and returns results to AI
5. AI formulates a response and streams it back to the user
6. All messages and tool calls are persisted in the database

